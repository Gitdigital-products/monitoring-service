# Monitoring Service

The monitoring-service is responsible for:
- Collecting health and performance metrics
- Tracking uptime and error rates
- Providing real-time monitoring endpoints
- Triggering alerts when thresholds are exceeded

## Features
- REST API with FastAPI
- Prometheus metrics endpoint
- InfluxDB for time-series data
- Webhook/Slack alert integration
- Auto-generated Swagger documentation
